CSS新属性之一contain

介绍

这个css新属性主要是让开发者更加灵活地掌握浏览器样式、布局和渲染工作，提前给浏览器一个先知。主要目的是隔离指定内容的样式、布局和渲染，它能新建一个边界，产生一个新的根布局，从而保证了它和它的子元素的DOM变化不会触发父元素重新布局和渲染等。

出现的原因

当我们对一个页面进行布局时，性能瓶颈通常是style、layout、paint。一般情况下，浏览器会把整个DOM作为CSS布局的上下文，因此：当我们改变部分DOM样式时，也会影响到其他部分，可能这些部分是不可见的，或者无需变化的，但是浏览器并不会知道，因此我们就需要一个属性来告知浏览器哪些内容应该在渲染范围内，哪些内容不在渲染范围内，那么contain就此产生的。

语法格式如下：

contain: none | strict | content | [ size || layout || style || paint ]

contain一共有7种不同的值

  • none： 默认值。使用此值不应用限制效果。
  • layout： 开启布局限制。即任何外部因素都不能影响其内部布局，反之亦然。例如：如果你改变一个元素的left元素，那么DOM中的每个元素可能需要进行检查。
  • style： 开启样式限制。因此，对元素及其后代可能产生影响的属性不会影响这个元素之外的任何内容。
  • paint： 开启渲染限制。这意味着元素的后代不会显示在其边界之外，例如：如果一个元素在屏幕外（或不可见），它的所有元素都是屏幕之外（或不可见）。如移动设备的隐藏菜单等。
  • size： 开启size限制。这意味着元素可以无需检查其后代节点进行布局，或者修改其尺寸不会影响到父元素的尺寸。
  • content： 开启除了size之外的所有限制
  • strict： 开启所有限制

应用场景

  • 页面小饰件（如固定的侧边菜单栏，分享工具等等）
  • 屏幕外的内容

限制条件

目前仅chrome52+和Opera40+支持
